# Vault - 기능 명세서

이 문서에서의 기능 이름들은 추후 변경이 가능하며 확정된 이름이 아닙니다.

---

## 기능 상세

### F-001: 통장

**상태**: 미구현

#### 개요

현금이나 통장 등을 등록하고 잔액을 한눈에 볼 수 있는 기능

#### 상세 설명

통장 추가 버튼을 통해 추가 가능. 이때 이름과 잔액, 사용가능 통화를 입력한 후 추가.
통화 변경 기능을 통해 한 통장 내에서 여러 통화를 지원할 수 있도록 하며, 기본 통화를 선택할 수 있도록 한다.
즐겨찾기를 지정하여 주 통장을 설정하고 이후 여러 기능들에서 기본값으로 이용할 수 있도록 한다.
이체 기능을 통해 통장 간에 돈을 간단하게 옮길 수 있는 시스템을 구현한다.
이체 기능에서는 통장에서 메뉴버튼을 눌러 이체를 누르게 되면 이체할 통장, 사용할 통화(이체할 통장에서 지원하는 통화중에 선택) 입력 후 이체 진행.
만약 이체 기능에서 사용할 통화가 이체 출발 통장에서 지원하지 않는 통화라면 어떤 통화 얼마를 어떤 통화 얼마로 변환하여 이체할지 입력받아 진행.
이체시 거래내역 자동 추가.
현재 등록된 통장들의 이름과 잔액을 실시간으로 조회할 수 있도록 한다.
자동결제를 통해서 변동예정사항이 생길 경우 해당 통장의 1달 후 변동 예정 금액을 연한 글씨로 하단에 표시한다.
주 사용 통화를 선택하여 총 자산 계산이나 통화 미선택시 기본값으로 사용하도록 한다.
삭제나 수정 기능을 추가하여 구현한다.
즐겨찾기 된 통장은 모든 거래내역 생성에서 통장 선택 드롭다운에서 가장 상단에 위치하도록 하여 쉽게 선택 가능하도록 한다.

### F-002: 거래내역

**상태**: 미구현

#### 개요

거래 내역을 열람하고 새로 생성할 수 있는 기능

#### 상세 설명

기간 선택하여 해당 기간의 거래내역 조회 기능. 기본적으로 1달 단위(이번달을 메인으로)로 표시. 버튼을 통해 표시되는 달 변경 가능(이번달 표시, 이번달로 복귀 버튼)
이체나 정기결제 등 자동 진행되는 거래들의 거래내역 자동 추가.
상단에 표시중인 기간의 총 수입/지출 표시
표시 시점 기준 아직 미실시된 거래내역은 연한 글씨로 표시. 상단에 표시되는 총 수입/지출에서도 예정 사항을 하단에 작은 글씨로 표기.
거래내역 수정 기능 구현. 정기 결제 수정시에는 해당 회차만 수정되도록 할지 앞으로의 모든 결제를 수정할지(이 옵션 선택시 정기결제 탭에서도 해당 항목 결제 금액 수정) 선택. 이체 내역 수정시 이체 내역 등록창과 유사한 창 띄운 후 수정시 생성되어 있는 이체 거래내역 2개 모두 자동 수정.
거래내역 생성 기능 구현. 이름(생략 가능. 생략시 수입 or 지출을 이름으로), 일자(미선택시 생성일자), 통장(미선택시 즐겨찾기 통장, 선택시 해당 통장에서 거래 가장 많은 통화 자동으로 통화 탭에 선택되도록), 통화(미선택시 주 사용 통화. 즐겨찾기 통장에서 주 사용 통화 지원 안할 경우 즐겨찾기 통장의 통화 중 거래내역 가장 많은 통화 선택), 금액, 카테고리(미선택시 기본 카테고리로 선택), 예산 모금or사용 선택 가능.
필터 기능을 통해 카테고리별, 통장별, 통화별 등 선택된 분류별 거래내역 모아보기 가능.

### F-003: 정기 결제

**상태**: 미구현

#### 개요

주기적으로 일어나는 수입/지출을 관리하는 기능

#### 상세 설명

새 항목 추가시 이름(생략 가능. 생략시 수입 or 지출을 이름으로), 일자(미선택시 생성일자), 통장(미선택시 즐겨찾기 통장, 선택시 해당 통장에서 거래 가장 많은 통화 자동으로 통화 탭에 선택되도록), 통화(미선택시 주 사용 통화. 즐겨찾기 통장에서 주 사용 통화 지원 안할 경우 즐겨찾기 통장의 통화 중 거래내역 가장 많은 통화 선택), 금액, 카테고리(미선택시 기본 카테고리로 선택), 결제 간격(일, 주, 달, 년 선택 후 숫자 입력하여 주기 선택. 기본 숫자 1) 입력하여 새로운 정보 등록.
일자를 기준으로 정해진 결제 간격마다 거래내역을 생성(거래 내역에 표시. DB가 너무 많이 발생하는 경우를 예방하기 위해 미실시된 거래의 경우 계산을 통해 표시하는 방식 선택. 예를 들어 거래내역에서 다음달 내역을 표시시키면 다음달에 일어날 정기 결제를 예정사항으로 표시하는 형태).
수정/삭제시 수정/삭제를 한 시점을 기준으로 아직 미실시된 데이터들을 일괄 수정/삭제.
월말, 윤년에 대한 예외처리를 확실하게 진행할 것.
정렬 기능 구현. 일 평균 결제금액, 최근 결제, 다음 회차까지 남은 기간 등을 기준으로 오름차순, 내림차순으로 정렬하는 기능 지원. 기본적으로는 다음회차까지 남은 기간을 오름차순으로 정렬하여 표시.
표시되는 내역은 이름, 금액, 결제 간격, 다음 결제일, 일(일, 주, 월, 년 중에 사용자가 설정에서 선택할 수 있도록) 평균 결제 금액을 표기.
상단에 총 월 평균 거래 금액을 표시(수입과 지출 따로 표기)

### F-004: 설정

**상태**: 미구현

#### 개요

마이페이지, 메뉴, 테마, 친구 등의 설정들을 다룰 수 있는 탭.

#### 상세 설명

마이페이지에서는 프로필 이미지, 닉네임, 생년월일, 비밀번호 변경, 소셜 계정 연동, 회원 탈퇴 등의 기능을 지원. SSO 연동 진행하여 SSO 계정의 정보를 수정.
메뉴에서는 메뉴탭에 표시될 항목과 순서 등을 설정할 수 있도록 한다. 모바일에서는 하단에 5개의 탭(중앙은 대시보드를 고정)을 표기할 예정이므로 4개의 탭 선택하여 순서 지정 가능하도록 하는 기능도 추가.
테마에서는 향후 개발될 테마 선택 기능을 추가할 것. 아직 미개발된 기능이므로 만들어두고 숨겨두도록 한다.
크레딧에서는 향후 개발될 크레딧을 통한 결제 기능 추가할 것. 아직 미개발된 기능이므로 숨겨둔다.
친구 관리에서는 SSO 계정에서 다른 계정을 친구 추가하고 관리할 수 있도록 한다. 메신저를 보내는 등 간단한 소셜 기능을 추가한다.
카테고리에서는 수입과 지출의 카테고리를 따로 관리하며 각 항목의 카테고리를 추가/수정/삭제할 수 있도록 한다. 즐겨찾기를 통해 카테고리 선택 드롭다운에서 최상단에 위치시킬 카테고리를 선택 가능. 순서 변경을 통해 드롭다운에 표시될 카테고리 순서 변경 가능.
이외 설정들로는 Vault 프로그램 내에서 사용자가 입력해둘 개인 설정사항들을 선택하도록 한다. 예를 들어 정기 결제 항목의 평균 금액을 얼마 단위로 표기할지, 주 사용 통화가 무엇인지 등.

### F-005: 예/적금

**상태**: 미구현

#### 개요

예금이나 적금, 자유적금, 청약 등을 쉽게 관리할 수 있는 기능

#### 상세 설명

예금 - 이름, 시작일자(미선택시 입력 일자), 기간(개월), 이자율, 통장(기본값 미선택. 미선택시 따로 거래내역 생성하지 않음. 선택시 해당 통장에서 입력한 금액만큼 거래내역 생성되도록 설정. 이때 거래 일자는 시작일자로 생성되도록), 금액, 비과세 체크박스(기본값 과세)
-> 이름, 만기일, 만기 수령액, 남은 일 수, 원금, 이자율 표시
적금 - 이름, 시작일자(미선택시 입력 일자), 기간(개월), 이자율, 통장(기본값 미선택. 미선택시 따로 거래내역 생성하지 않음. 선택시 해당 통장에서 입력한 금액만큼 거래내역 생성되도록 설정. 이때 거래 일자는 매 적금 납입일마다 생성되도록.), 금액(회차당 납입금. 만약 입력된 시작일자가 과거여서 이미 적금에 돈이 있다면 현재 적금에 들어있는 돈을 입력받는 탭을 추가 생성하도록.), 비과세 체크박스(기본값 과세)
-> 이름, 만기일, 만기 수령액, 남은 일 수, 원금, 이자율 표시
자유적금 - 이름, 시작일자(미선택시 입력 일자), 기간(개월), 이자율, 금액(시작일자가 과거라면 입력창을 표시하고 미래라면 표시하지 않음. 입력받은 값은 지금까지 자유적금에 납입한 원금 총액), 비과세 체크박스(기본값 과세)
-> 이름, 만기일, 만기 수령액, 남은 일 수, 원금, 이자율 표시. 추가 버튼 만들어 납입금 추가 가능. 추가시 통장, 금액 입력받아 납입금 추가하는 방식. 이때 거래내역을 자동으로 생성.
청약 - 이름, 시작일자(미선택시 입력 일자), 이자율, 통장(기본값 미선택. 미선택시 따로 거래내역 생성하지 않음. 선택시 해당 통장에서 입력한 금액만큼 거래내역 생성되도록 설정. 이때 거래 일자는 매 청약 납입일마다 생성되도록.), 금액(회차당 납입금. 만약 입력된 시작일자가 과거여서 이미 적금에 돈이 있다면 현재 청약에 들어있는 돈을 입력받는 탭을 추가 생성하도록.), 비과세 체크박스(기본값 과세)
-> 이름, 원금, 이자율 표시

### F-006: 통계

**상태**: 미구현

#### 개요

현재 사용자의 거래 내역의 변천사나 총 자산의 변동 등을 조회할 수 있는 기능

#### 상세 설명

아직 완벽한 구상이 끝나지 않음.
그래프를 통해 달별로 수입/수출/총 자산 변동폭 표시.
총 자산의 변동을 그래프를 통해 볼 수 있도록.
자신의 자산 보유 현황을 분류별로 볼 수 있도록.
카테고리별 거래 내역을 확인할 수 있도록.

### F-007: 예산

**상태**: 미구현

#### 개요

사용자가 예산 규모를 설정하고 이를 채우거나 소비할때 관리할 수 있도록 하는 기능.

#### 상세 설명

아직 완벽한 구상이 끝나지 않음.
예산을 생성할 시 이름과 금액을 입력받아 생성.
예산에서 납입 버튼을 통해 지금 보유 자산에서 예산으로 저장 가능.
이후 거래내역 추가시 예산에서 사용하며 사용 규모 추정할 수 있도록 하는 기능.

### F-008: 대시보드

**상태**: 미구현

#### 개요

공지사항이나 주요 통계, 보유 자산, 최근 거래내역(수입/지출)을 한눈에 보기 쉽게 하도록 하는 메인 기능.

#### 상세 설명

아직 완벽한 구상이 끝나지 않음.
의견 나누며 추가 구상 계획.

### F-009: 공지사항

**상태**: 미구현

#### 개요

관리자가 등록한 공지사항을 열람할 수 있도록 하는 기능.

#### 상세 설명

관리자는 공지사항을 등록, 수정할 수 있도록 한다.
관리자가 상단 공지 게시글을 최대 3개까지 지정할 수 있도록 한다.
사용자들은 관리자가 등록한 공지를 보고 댓글을 작성할 수 있도록 한다. 좋아요 표시도 지원.

### F-010: 휴지통

**상태**: 미구현

#### 개요

삭제된 항목들을 모아두고 복원할 수 있도록 하는 기능.

#### 상세 설명

삭제된 항목들을 삭제된 일자가 최신인 것들부터 볼 수 있도록 한다.
복원 기능을 통해 삭제된 내역을 다시 복구할 수 있도록 한다.

### F-011: 문의

**상태**: 미구현

#### 개요

관리자와 사용자가 소통을 하며 문의사항을 문의할 수 있는 기능.

#### 상세 설명

사용자가 문의사항을 등록할 경우 관리자가 이를 열람하고 답변할 수 있는 기능.
캡처한 사진을 첨부할 수 있도록 하고 자세한 내용을 적어줄수록 유리하다는 내용 표시.
관리자가 답변한 이후 사용자가 해당 문의 열람시 답변을 보고 유용, 유용하지 않음 표시 가능하도록.
